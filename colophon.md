  <h1 class="post-header__title">Colophon</h1>
  <p class="post-header__lede">How this site is made, and why it looks the way it does.</p>
</div>

<div class="post-content">

  <div class="para-row">
    <div class="para-body">
      <h3>✦ &ensp; The Platform &ensp; ✦</h3>
      <p>Built with <a href="https://jekyllrb.com">Jekyll</a> and hosted on <a href="https://pages.github.com">GitHub Pages</a>. No JavaScript frameworks, no build pipeline beyond Jekyll's own, no npm. The source is a folder of Markdown files and some Sass. This was a deliberate choice and one I remain pleased with.</p>
    </div>
    <div class="para-note">
      <div class="mn">
        <span class="mn__mark">—</span>
        The absence of a JavaScript framework is not a political statement. It is a preference for the quiet life.
      </div>
    </div>
  </div>

  <div class="para-row">
    <div class="para-body">
      <h3>✦ &ensp; Typography &ensp; ✦</h3>
      <p>Body text is set in <a href="https://fonts.google.com/specimen/EB+Garamond">EB Garamond</a>, a free revival of Claude Garamond's sixteenth-century typefaces. Display headings use <a href="https://fonts.google.com/specimen/Cormorant+Garamond">Cormorant Garamond</a>, a more refined interpretation of the same historical source. Metadata, dates, tags, and all the small machinery of the site is set in <a href="https://fonts.google.com/specimen/JetBrains+Mono">JetBrains Mono</a>.</p>
      <p>The drop cap on each post's opening paragraph is set in Cormorant Garamond in the site's red accent, because small pleasures matter.</p>
    </div>
    <div class="para-note">
      <blockquote><p>"The detail is not the detail. It makes the design."</p></blockquote>
    </div>
  </div>

  <div class="para-row">
    <div class="para-body">
      <h3>✦ &ensp; Colour &ensp; ✦</h3>
      <p>The palette is warm paper, ink, and one red accent. The background is not white — it is <code>#f6f1e7</code>, which is the colour of a page that has been handled. The red is <code>#c84b2f</code>, used only where it earns its place: drop caps, footnote markers, hover states. Everything else is various shades of ink.</p>
    </div>
    <div class="para-note">
      <div class="mn">
        <span class="mn__mark">—</span>
        Pure white backgrounds cause a very faint eye strain over long reading sessions. This is not a medical claim.
      </div>
    </div>
  </div>

  <div class="para-row">
    <div class="para-body">
      <h3>✦ &ensp; The Margin Notes &ensp; ✦</h3>
      <p>Each paragraph in a post can have an aligned margin note beside it — an aside, a clarification, or a confession. These are set using a <code>.para-row</code> CSS grid that keeps the note flush beside its anchor paragraph. No JavaScript, no absolute positioning. Each note sits next to the text it annotates because they are in the same grid row. This is the entire trick.</p>
    </div>
    <div class="para-note">
      <div class="mn">
        <span class="mn__mark">—</span>
        The margin note system was the reason for building a custom theme in the first place. Everything else followed from it.
      </div>
    </div>
  </div>

</div>
